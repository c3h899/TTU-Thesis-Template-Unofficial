A LaTeX thesis-dissertation style/template for Texas Tech

The Graduate School at Texas Tech has changed their thesis and dissertation formatting guidelines. The current LaTeX thesis style floating around the department was written in the mid 90s and is out of date. When I taught a course in LaTeX in 2007, I decided to revise the thesis style to meet the new guidelines. It has since been successfully used to submit theses and dissertations by a number of people.

If you use this style, I would appreciate knowing about any issues you encounter, as well as suggestions for improvement.

There are some user contributed notes at the end of this page.

Files needed

You will need to download at least two files (see below). The first file is a LaTeX documentclass file called ttuthes2007.cls. The second file is the actual template itself: thesis_template.tex. This document style uses two LaTeX packages which you may or may not have: fancyhdr.sty and setspace.sty. If you are using MiKTeX on a computer on which you have an administrator account, these packages will be downloaded and installed automatically for you. Otherwise, you may have to download them explicitly.

Download instructions

Create a working folder for your thesis. (Creating one on your TechDrive is a good choice.) Download the following files into this folder. (You will probably have to right-click on each file.)

  • ttuthes2007.cls (required!)
  • thesis_template.tex (required!)
  • fancyhdr.sty (only required if you don't already have this package.)
  • setspace.sty (only required if you don't already have this package.)

Instructions

You may wish to rename thesis_template.tex to something like mythesis.tex. This will be the root file for your thesis. Open this in your editor or TeXnicCenter and start writing. For best results, carefully follow the instructions given as comments.

User notes:

From Roger Coffey: I found how to correct the A4 sizing problem using MikTex. I'm using the TeXnicCenter front end and noticed that if I change something like the \paperheight to 17.0in, LaTex would send the text off the end of the paper. It appeared that LaTex was working properly but the pages were ending up the A4 size.

I'm using the compile method of LaTex=>PS=>PDF. In TeXnicCenter, follow the menu path of: Build->Define Output Profiles

Next click the "Postprocessor" tab. Under "Profiles" I have "LaTex=>PS=>PDF selected. In the adjacent box I have the following lines:
DviPs (PDF)
Ghostscript(ps2pdf)

Click on the "Ghostscript(ps2pdf)" line. For the scripts for "Arguments:" change the

-sPAPERSIZE=a4 to -sPAPERSIZE=letter

I'm not sure why these arguments are set this way but they are. It appears that the last Ghostscript step was undoing all of the "Letter" sizing and converting it to "A4".

Now I can go into Adobe, click:

File->Document Properties

...and it shows 8.50 in x 11.00 in
Let me know if this is any help.
Good Luck!
Roger
